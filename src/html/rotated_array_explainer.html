The ONE question that solves everything
🔑 "Which half am I guaranteed sorted?" When you pick mid — one of the two halves (lo→mid or mid→hi) is always guaranteed to be sorted. Figure out which one, then check if your target lives there.
How to tell which half is sorted:

This is why a[lo] ≤ a[mid] is the core condition of every rotated array solution — it tells you where order is guaranteed.
⚠️ Common mistake: Beginners try to find the pivot first. You don't need to! Just check which half is sorted, then eliminate the other half if target isn't in the sorted one.

def search(nums, target): lo, hi = 0, len(nums) - 1 while lo <= hi: mid = (lo + hi) // 2 if nums[mid] == target: return mid # ── Step 1: Is LEFT half sorted? ────────────────── if nums[lo] <= nums[mid]: # Step 2: Is target inside the sorted left half? if nums[lo] <= target < nums[mid]: hi = mid - 1 # target in left → search left else: lo = mid + 1 # target not in left → search right # ── Step 3: RIGHT half must be sorted ───────────── else: # Step 4: Is target inside the sorted right half? if nums[mid] < target <= nums[hi]: lo = mid + 1 # target in right → search right else: hi = mid - 1 # target not in right → search left return -1

Variant: Find Minimum (LC #153) — different logic, same insight
Here you're not searching for a target. You're searching for the pivot — the point where the array "drops". The insight: if a[mid] > a[hi], the pivot is on the right side.
def findMin(nums): lo, hi = 0, len(nums) - 1 while lo < hi: # note: lo < hi, not lo <= hi mid = (lo + hi) // 2 if nums[mid] > nums[hi]: # mid is in the LEFT (higher) sorted half # pivot must be to the RIGHT of mid lo = mid + 1 else: # mid is in the RIGHT (lower) sorted half # pivot could BE mid, so don't skip it hi = mid return nums[lo] # lo == hi == index of minimum
⚠️ Why lo < hi instead of lo <= hi? Because we never want to go past the answer. When lo==hi, we're AT the minimum.
a[mid] > a[hi]
Left half is "the tall part" → minimum is on RIGHT → lo = mid + 1
a[mid] ≤ a[hi]
Right half is "the short part" → minimum is on LEFT or IS mid → hi = mid
🧠 Memory trick: "tall side → go right". The minimum always lives at the bottom of the drop. If mid is on the tall (left) hill, slide right.
Cheat sheet — stick this to your wall
Before you write a single line of code, answer: Am I (A) searching for a value, or (B) finding the minimum/pivot?
(A) Search for target • Use lo ≤ hi
• Check: a[lo] ≤ a[mid]
• Find which half is sorted
• Check if target fits in it

(B) Find minimum • Use lo < hi

3 tips to stop forgetting while solving
🖊️
Write the question first. Before code, write: "Which half is sorted?" on paper. This forces the right mental model every time.
🔢
Dry-run with [4,5,6,7,0,1,2]. This specific array has 4 elements on the left hill and 3 on the right. Use it for every practice run.
🧩
Don't find pivot explicitly. Many students waste 10 min finding the pivot index first. You never need it — just compare a[lo], a[mid], a[hi].
🔁
Solve LC #153 → LC #33 in that order. Finding minimum builds the intuition; searching uses it. Never do them in reverse.
⚠️ Duplicates (LC #81): When a[lo] == a[mid] == a[hi] you can't tell which half is sorted. Only safe move: lo++, hi--. This makes worst case O(n) — that's expected and correct.
